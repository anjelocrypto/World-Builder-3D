Build an original browser-based 3D multiplayer open-world city sandbox prototype inspired by modern open-world action/driving games, but do NOT use GTA, Rockstar, Los Santos, real GTA assets, logos, music, characters, maps, UI, or copyrighted material.

Goal: create a playable MVP, not a full AAA game.

Tech stack:
- React + Vite + TypeScript
- Three.js / @react-three/fiber / @react-three/drei for 3D
- @react-three/rapier or cannon-es for physics
- Node.js + Express backend
- Socket.io for realtime multiplayer

Core features:
1. 3D city test map with roads, sidewalks, simple buildings, lights, ramps, and spawn points.
2. Third-person player controller with walking, running, jumping, camera follow, and collision.
3. Drivable vehicles with enter/exit interaction, acceleration, braking, steering, basic suspension feel, and camera switching.
4. Multiplayer:
   - Players can join the same world.
   - Show other players in realtime with name labels.
   - Sync position, rotation, movement state, and vehicle state.
   - Use server-authoritative validation where practical.
5. Simple lobby:
   - Player enters username.
   - Join world button.
   - Show connected player count.
6. Basic UI:
   - Health bar
   - Minimap placeholder
   - Speedometer when driving
   - Interaction prompt when near vehicle
7. Gameplay loop:
   - Spawn in city
   - Explore on foot
   - Enter/exit cars
   - Drive around with other online players
   - Add simple checkpoint race mission as first activity
8. Code quality:
   - Organize code into clear folders: client, server, shared.
   - Add comments only where logic is complex.
   - Keep assets lightweight and procedural/simple so it runs well on Replit.
   - Include setup instructions in README.
   - Make sure `npm install` and `npm run dev` start both frontend and backend.

Important constraints:
- This is an original prototype, not a GTA clone.
- Use placeholder geometry and free/simple assets only.
- Prioritize a stable playable multiplayer demo over visual polish.
- If something is too large, implement the smallest working version first and leave TODOs for expansion.

Start by scaffolding the project, then implement the multiplayer walking prototype, then vehicles, then city environment, then UI and checkpoint race.
